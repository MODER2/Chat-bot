# Пошаговая инструкция: от пустого компьютера до запущенного чат-бота

Здесь расписан **каждый шаг**: что установить, как открыть/собрать проект, как
запустить и что делать при ошибках. Есть два пути:

- **Путь A (рекомендуется)** — открыть уже готовый проект из этого репозитория.
- **Путь B** — создать проект с нуля своими руками (чтобы понять, откуда берётся
  каждый файл).

---

## Часть 0. Что нужно установить

1. **JDK 17 или новее** (Java Development Kit). Проверьте в терминале:
   ```bash
   java -version
   ```
   Должно показать версию 17+. Если Java нет — поставьте, например,
   [Eclipse Temurin 17](https://adoptium.net/) или OpenJDK.

2. **IntelliJ IDEA** — среда разработки. Подойдёт бесплатная **Community Edition**:
   <https://www.jetbrains.com/idea/download/>.

3. **Maven** ставить отдельно НЕ обязательно — он встроен в IntelliJ IDEA.

> JavaFX отдельно скачивать НЕ нужно: он подключается автоматически как
> зависимость Maven (см. файл `pom.xml`).

---

## Часть A. Открыть готовый проект (быстрый путь)

1. Скачайте проект:
   ```bash
   git clone https://github.com/MODER2/Chat-bot
   ```
2. В IntelliJ IDEA: **File → Open…** и выберите папку проекта (ту, где лежит
   `pom.xml`).
3. IntelliJ увидит `pom.xml` и предложит загрузить зависимости — согласитесь
   («Load Maven Project» / «Trust Project»). Дождитесь, пока внизу исчезнёт
   индикатор загрузки (IDEA скачает JavaFX).
4. Запуск — см. **Часть C**.

---

## Часть B. Создать проект с нуля (подробно, файл за файлом)

### Шаг 1. Создать Maven-проект

1. **File → New → Project…**
2. Слева выберите **Maven** (или «New Project» → тип Maven). Снимите галочку
   с архетипов (archetype) — делаем пустой проект.
3. Задайте:
   - **Name:** `chatbot`
   - **GroupId:** `com.example`
   - **ArtifactId:** `chatbot`
   - **JDK:** 17 (или новее).
4. Нажмите **Create**. IntelliJ создаст папку проекта и файл `pom.xml`.

### Шаг 2. Заполнить `pom.xml`

Откройте `pom.xml` в корне проекта и впишите зависимости JavaFX и плагины.
Возьмите готовое содержимое из файла [`pom.xml`](pom.xml) этого репозитория
(там подробные комментарии). Ключевое в нём:
- версия Java — 17;
- зависимости `javafx-controls` и `javafx-graphics`;
- плагин `javafx-maven-plugin` с указанием главного класса
  `com.example.chatbot.ui.ChatApplication`.

После изменения `pom.xml` нажмите значок Maven «Reload» (или **Ctrl+Shift+O**),
чтобы IntelliJ скачал библиотеки.

### Шаг 3. Создать структуру папок и классов

В IntelliJ правой кнопкой по `src/main/java` → **New → Package**, создайте пакеты
и в них классы. Должна получиться такая структура (полный список файлов):

```
chatbot/
├── pom.xml                         ← настройки сборки и зависимости
├── src/main/java/com/example/chatbot/
│   ├── model/
│   │   ├── Author.java             ← enum: USER / BOT
│   │   ├── Message.java            ← одно сообщение (автор, текст, время)
│   │   └── ChatHistory.java        ← история в оперативной памяти + статистика
│   ├── bot/
│   │   ├── IBot.java               ← интерфейс бота
│   │   ├── AbstractBot.java        ← абстрактный класс с общим алгоритмом
│   │   ├── SimpleBot.java          ← конкретный бот (список обработчиков)
│   │   ├── BotContext.java         ← доступ к истории и имени пользователя
│   │   └── handler/
│   │       ├── MessageHandler.java ← абстрактный обработчик (база, regex)
│   │       ├── GreetingHandler.java   ← приветствия
│   │       ├── TimeHandler.java       ← «который час?»
│   │       ├── DateHandler.java       ← «какое сегодня число?»
│   │       ├── CalculatorHandler.java ← «умножь 12 на 157» (параметры)
│   │       ├── StatsHandler.java      ← «сколько сообщений?»
│   │       ├── NameHandler.java       ← «как меня зовут?»
│   │       ├── HelpHandler.java       ← «помощь»
│   │       ├── WebFactHandler.java    ← факт из интернета (веб-запрос)
│   │       └── LlmHandler.java        ← вопрос к нейросети (LLM)
│   ├── persistence/
│   │   ├── HistoryStorage.java     ← интерфейс хранилища
│   │   └── FileHistoryStorage.java ← запись/чтение файла истории
│   └── ui/
│       ├── ChatApplication.java    ← точка входа (main), «дирижёр»
│       ├── LoginWindow.java        ← окно ввода имени
│       └── ChatWindow.java         ← основное окно чата
└── src/main/resources/com/example/chatbot/ui/
    ├── icon.png                    ← иконка приложения
    └── styles.css                  ← шрифт и цветовая палитра
```

> **Рекомендуемый порядок написания** (от простого к сложному, чтобы код
> компилировался по ходу): сначала `model`, затем `bot/handler` →
> `bot` (IBot, AbstractBot, SimpleBot), потом `persistence`, и в конце `ui`.

Содержимое каждого файла берите из этого репозитория — весь код подробно
прокомментирован построчно. Не забудьте про папку **resources**: файлы `icon.png`
и `styles.css` должны лежать именно в
`src/main/resources/com/example/chatbot/ui/` — программа ищет их по этому пути.

### Шаг 4. Проверить, что всё на месте

В терминале IntelliJ (вкладка **Terminal** внизу) выполните:
```bash
mvn clean compile
```
Должно закончиться строкой **BUILD SUCCESS**. Если есть ошибки — читайте, в каком
файле и на какой строке, и сверяйтесь с исходником.

## Часть C. Как запустить программу

### Способ 1 — через Maven (самый надёжный)

В терминале в корне проекта:
```bash
mvn clean javafx:run
```
Maven сам подставит модули JavaFX и запустит приложение. Появится окно ввода
имени, затем — чат.

### Способ 2 — кнопкой «Run» в IntelliJ

1. Откройте `ChatApplication.java`.
2. Нажмите зелёный треугольник ▶ слева от метода `main`.
3. Если IntelliJ ругается на отсутствие модулей JavaFX при прямом запуске —
   используйте Способ 1 (`mvn javafx:run`), он не требует ручной настройки
   `module-path`.

## Часть D. Как пользоваться ботом

После запуска введите имя и попробуйте команды:

| Что написать | Что сделает бот |
|--------------|-----------------|
| `Привет, Бот!` | поздоровается по имени |
| `Который час?` | покажет текущее время |
| `Какое сегодня число?` | покажет дату и день недели |
| `умножь 12 на 157` | посчитает: `12 * 157 = 1884` |
| `сложи 2 и 3`, `15 / 4` | другие примеры калькулятора |
| `статистика` | сколько было сообщений |
| `как меня зовут?` | назовёт ваше имя |
| `расскажи факт` | сходит в интернет за фактом |
| `ии: что такое JavaFX?` | спросит нейросеть (нужен ключ, см. ниже) |
| `помощь` | покажет все команды |

Горячие клавиши в поле ввода: **Enter** или **Ctrl+Enter** — отправить,
**Shift+Enter** — перенос строки.

История сохраняется в файл `chat_history.txt` рядом с программой при закрытии и
подгружается при следующем запуске.

### (Опционально) Включить ответы нейросети

Команда `ии: ...` работает, только если задан ключ доступа. Перед запуском
установите переменную окружения (ключ в код НЕ вписывается — это небезопасно):

```bash
# Linux / macOS:
export OPENAI_API_KEY="ваш_ключ"
mvn clean javafx:run
```
При желании можно поменять адрес сервера (`OPENAI_BASE_URL`) и модель
(`OPENAI_MODEL`) — см. комментарии в `LlmHandler.java`. Без ключа бот просто
сообщит, что функция выключена, — программа продолжит работать.

## Часть E. Возможные проблемы и решения

| Симптом | Причина | Решение |
|---------|---------|---------|
| `java: error: release version 17 not supported` | старый JDK | поставить JDK 17+ и выбрать его в **File → Project Structure → SDK** |
| При запуске: `Error: JavaFX runtime components are missing` | прямой запуск без module-path | запускать через `mvn javafx:run` |
| Кракозябры вместо русских букв | неверная кодировка | убедиться, что файлы в UTF-8 (в `pom.xml` уже задано `UTF-8`) |
| `расскажи факт` отвечает ошибкой сети | нет интернета | это нормально, бот не падает; проверьте подключение |
| Не видно иконку/стили | файлы не в `resources` | положить `icon.png` и `styles.css` в `src/main/resources/com/example/chatbot/ui/` |
