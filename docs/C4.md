# C4 — архитектура чат-бота

**C4** — это способ описывать архитектуру программы на четырёх уровнях
детализации (отсюда «C4»). Каждый следующий уровень — как увеличение масштаба
на карте: от общего вида к деталям.

1. **Context (Контекст)** — система целиком и кто/что с ней взаимодействует.
2. **Container (Контейнеры)** — крупные части системы (приложение, файл, внешние сервисы).
3. **Component (Компоненты)** — модули внутри приложения.
4. **Code (Код)** — конкретные классы (это уже показано в [UML.md](UML.md)).

> Диаграммы написаны на Mermaid. Если C4-синтаксис не отрисовался в вашем
> просмотрщике, ниже каждого блока есть пояснение словами.

---

## Уровень 1 — Context (Контекст)

Кто пользуется системой и с какими внешними сервисами она общается.

```mermaid
C4Context
    title Контекст системы «Чат-бот»

    Person(user, "Пользователь", "Человек, который общается с ботом")
    System(chatbot, "JavaFX Чат-бот", "Десктоп-приложение: отвечает на сообщения, считает, хранит историю")
    System_Ext(factApi, "API фактов", "catfact.ninja — публичный веб-сервис")
    System_Ext(llmApi, "LLM-сервер", "OpenAI-совместимый API (опционально)")

    Rel(user, chatbot, "Пишет сообщения, читает ответы")
    Rel(chatbot, factApi, "HTTP GET запрос за фактом")
    Rel(chatbot, llmApi, "HTTP POST вопрос к нейросети")
```

**Словами:** пользователь общается с приложением. Приложение при необходимости
ходит в интернет за фактом (catfact.ninja) и к LLM-серверу (если задан ключ).

## Уровень 2 — Container (Контейнеры)

Из каких крупных частей состоит сама система.

```mermaid
C4Container
    title Контейнеры системы «Чат-бот»

    Person(user, "Пользователь")

    Container_Boundary(app, "JavaFX Чат-бот") {
        Container(ui, "GUI (JavaFX)", "Java, JavaFX", "Окна: ввод имени и чат")
        Container(logic, "Логика бота", "Java", "Распознаёт команды и формирует ответы")
        Container(storage, "Хранилище истории", "Java + файл", "Сохраняет и читает переписку")
    }

    ContainerDb(file, "chat_history.txt", "Текстовый файл", "Хранит историю между запусками")
    System_Ext(factApi, "API фактов")
    System_Ext(llmApi, "LLM-сервер")

    Rel(user, ui, "Вводит сообщения")
    Rel(ui, logic, "Передаёт текст, получает ответ")
    Rel(ui, storage, "Сохранить/загрузить историю")
    Rel(storage, file, "Чтение/запись (UTF-8)")
    Rel(logic, factApi, "HTTP GET")
    Rel(logic, llmApi, "HTTP POST")
```

**Словами:** внутри приложения три части — интерфейс (окна), логика бота и
хранилище. Хранилище работает с текстовым файлом, логика — с внешними API.

## Уровень 3 — Component (Компоненты)

Что находится внутри контейнера «Логика бота».

```mermaid
C4Component
    title Компоненты контейнера «Логика бота»

    Container(ui, "GUI (JavaFX)", "Java", "Окно чата")

    Component(ibot, "IBot / AbstractBot / SimpleBot", "Java", "Перебирает обработчики и выбирает ответ")
    Component(context, "BotContext", "Java", "Доступ к истории и имени пользователя")
    Component(handlers, "MessageHandler + наследники", "Java + regex", "Каждый отвечает за свою команду")

    Rel(ui, ibot, "respond(текст)")
    Rel(ibot, handlers, "canHandle / respond")
    Rel(ibot, context, "Передаёт обработчикам")
    Rel(handlers, context, "Читают историю и имя")
```

**Словами:** окно вызывает `IBot.respond()`. Бот по очереди спрашивает у
обработчиков, кто умеет ответить. Обработчики при необходимости берут данные из
`BotContext`.

## Резюме архитектурных решений

- **Разделение на слои** (UI / логика / хранение) — каждый слой можно менять
  независимо.
- **Программирование на интерфейсах** (`IBot`, `HistoryStorage`) — реализации
  легко заменяются.
- **Паттерн «Стратегия» через обработчики** (`MessageHandler`) — поведение бота
  расширяется без изменения существующего кода.
