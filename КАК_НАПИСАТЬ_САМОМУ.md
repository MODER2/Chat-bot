# Как написать чат-бота самому: пошаговый план разработки

Это **методичка по процессу**: не «возьми готовый код», а «вот в каком порядке
думать и писать, чтобы собрать бота своими руками». Учтены твои условия:

- **без LLM и без веб-запросов** — только заранее заданные «команды»;
- **UI делаем в последнюю очередь** через **Scene Builder (FXML)**;
- **C4 не нужен**.

> В этом репозитории уже лежит рабочий пример, но там интерфейс собран кодом
> (без FXML). Ниже — как сделать тот же проект, но с UI на Scene Builder, и в той
> последовательности, в какой удобно писать самостоятельно. Логика (модель, бот,
> хранение) одинаковая в обоих подходах — меняется только способ создания окон.

---

## Главный принцип: «снизу вверх, UI — в конце»

Программу пишут не в том порядке, в каком пользователь её видит. Видит он сначала
окно, а пишем мы окно последним. Почему:

1. **Окну нужно, что показывать.** Сначала должны существовать данные (модель) и
   тот, кто формирует ответы (бот). Иначе окно нечем наполнять.
2. **Логику можно проверить без окна.** Бота легко протестировать в консоли — это
   быстрее, чем каждый раз запускать GUI.
3. **Так требует задание.** «Логику реализуйте в отдельном модуле, независимом от
   интерфейса». Если писать UI последним, невозможно случайно «впихнуть» логику в
   окно — она физически написана раньше и отдельно.

Поэтому порядок слоёв такой:

```
1) MODEL        — данные (сообщение, история)
2) PERSISTENCE  — сохранение истории в файл
3) BOT (логика) — распознавание команд и ответы   ← ядро задания
4) ПРОВЕРКА     — временный консольный main без окон
5) UI (FXML)    — окна в Scene Builder              ← в самом конце
```

---

## Этап 0. Подготовка (один раз)

1. Установить **JDK 17+** и **IntelliJ IDEA**.
2. Установить **Scene Builder** — отдельная программа для рисования FXML мышкой:
   <https://gluonhq.com/products/scene-builder/>. В IntelliJ потом укажешь путь к
   нему: **Settings → Languages & Frameworks → JavaFX → Path to SceneBuilder**.
3. Создать **Maven-проект** и в `pom.xml` подключить JavaFX
   (`javafx-controls`, `javafx-fxml`) и плагин `javafx-maven-plugin`.
   Обрати внимание: для FXML нужен именно модуль **`javafx-fxml`** (в чисто
   программном UI он не требовался).

> Дальше всё пишем в пакете `com.example.chatbot` и его подпакетах.

## Этап 1. MODEL — данные (пиши первым)

Это самые простые классы, с них начинают, потому что от них зависит всё остальное.

### Файл 1: `model/Author.java` — кто автор сообщения

Перечисление (`enum`) из двух значений. Почему enum, а не строка: значений всего
два, и компилятор не даст опечататься.

```java
public enum Author { USER, BOT }
```

### Файл 2: `model/Message.java` — одно сообщение

Хранит **три вещи, которые требует задание**: автора, текст и **время отправки**.
Сделай поля `final` (неизменяемый объект — безопаснее).

Ключевые мысли:
- конструктор, который сам ставит время `LocalDateTime.now()` — для новых сообщений;
- метод `toLine()` → строка для файла (`USER|2025-05-31 14:03:17|Привет`);
- статический `fromLine(String)` → обратно собрать `Message` из строки файла.

Эти два метода понадобятся на Этапе 2 (сохранение). Пишешь их сразу — потом не
придётся возвращаться.

### Файл 3: `model/ChatHistory.java` — история в оперативной памяти

Обёртка над `List<Message>`. Требование задания: «хранить историю в оперативной
памяти». Что умеет:
- `add(Message)` — добавить;
- `getMessages()` — отдать список (лучше «только для чтения», через
  `Collections.unmodifiableList`, чтобы снаружи не сломали);
- счётчики для статистики: всего сообщений, от пользователя, от бота — это нужно
  для команды «сколько сообщений?».

**Проверка этапа:** код должен компилироваться. Запускать пока нечего.

## Этап 2. PERSISTENCE — сохранение в файл

Требование: «записывать историю в файл при завершении и загружать при запуске».
Выноси это в отдельный класс, чтобы UI потом просто вызвал два метода.

### Файл 4: `persistence/HistoryStorage.java` — интерфейс

```java
public interface HistoryStorage {
    void save(List<Message> messages);
    List<Message> load();
}
```
Зачем интерфейс: сегодня файл, завтра, может, база — и остальной код не изменится.

### Файл 5: `persistence/FileHistoryStorage.java` — реализация на файле

- `save()` — каждое сообщение через `Message.toLine()` пишем строкой в файл
  (`Files.write`, кодировка **UTF-8** — иначе русский текст «поломается»).
- `load()` — если файла нет, вернуть пустой список; иначе прочитать строки и
  собрать `Message.fromLine()`.

**Проверка этапа:** можно во временном `main` создать пару `Message`, сохранить,
потом загрузить и распечатать — убедиться, что файл пишется и читается.

## Этап 3. BOT — логика (ядро задания)

Здесь главное: **сделать логику гибкой и независимой от окон**. В этом пакете не
должно быть ни одного `import javafx...`.

Идея архитектуры: бот не должен быть одним гигантским `if-else`. Вместо этого —
**список «обработчиков»**, где каждый отвечает за одну тему (приветствие, время,
калькулятор…). Добавить новую команду = добавить новый класс, не трогая старые.

### Файл 6: `bot/handler/MessageHandler.java` — абстрактный обработчик

Задание прямо требует «использовать абстрактные классы для описания программного
интерфейса». Вот он. Общая часть (работа с регуляркой) — здесь, а наследники
дописывают только ответ.

```java
public abstract class MessageHandler {
    private final Pattern pattern;

    protected MessageHandler(String regex) {
        // CASE_INSENSITIVE + UNICODE_CASE — чтобы «Привет» = «привет» и работала кириллица
        this.pattern = Pattern.compile(regex,
                Pattern.CASE_INSENSITIVE | Pattern.UNICODE_CASE);
    }

    /** Подходит ли это сообщение данному обработчику? */
    public boolean canHandle(String input) {
        return input != null && pattern.matcher(input.trim()).find();
    }

    /** Доступ к группам захвата для наследников (числа в калькуляторе и т.п.). */
    protected Matcher matcher(String input) {
        return pattern.matcher(input.trim());
    }

    public abstract String getDescription();             // для команды «помощь»
    public abstract String respond(String input, ChatHistory history); // сам ответ
}
```

> Я передаю в `respond()` ещё и `ChatHistory` — он нужен обработчику статистики.
> Если хочешь чище — заведи маленький класс-контекст и передавай его. Для учебной
> работы достаточно передавать историю напрямую.

### Файлы 7…N: конкретные обработчики (по одному на тему)

Пиши их по очереди, от простого к сложному. Для каждого — свой regex-триггер и
текст ответа. Минимально нужны под требования задания:

**`GreetingHandler`** — реплики шаблона «Привет, Бот!»:
```java
super("\\b(привет\\w*|здравствуй\\w*|доброе\\s+(утро|день|вечер))\\b");
// respond: "Привет! Чем помочь?"
```

**`TimeHandler`** — простой вопрос «Который час?»:
```java
super("(который\\s+час|сколько\\s+времени|\\bвремя\\b)");
// respond: "Сейчас " + LocalTime.now().format(...);
```

**`StatsHandler`** — вопрос о статистике переписки:
```java
super("(статистик\\w*|сколько\\s+сообщений)");
// respond: использует history.getTotalCount() / getUserMessageCount() / getBotMessageCount()
```

**`CalculatorHandler`** — команда **с параметрами** «умножь 12 на 157». Здесь
ключевая работа с regex — **группы захвата** `( ... )`:
```java
// (глагол) (число1) на/и (число2)
super("(умнож\\w*|раздел\\w*|слож\\w*|вычт\\w*)\\s+(-?\\d+)\\s+(?:на|и|от)\\s+(-?\\d+)");

public String respond(String input, ChatHistory h) {
    Matcher m = matcher(input);
    m.find();
    String verb = m.group(1);            // "умножь"
    long a = Long.parseLong(m.group(2)); // 12
    long b = Long.parseLong(m.group(3)); // 157
    long result = verb.startsWith("умнож") ? a * b
                : verb.startsWith("слож")  ? a + b
                : verb.startsWith("вычт")  ? a - b
                : a / b;
    return a + " ? " + b + " = " + result;
}
```
Здесь видно, **как из текста достаются параметры**: круглые скобки в regex
«запоминают» куски, а `m.group(1/2/3)` их возвращают.

**`HelpHandler`** — «помощь»: проходит по всем обработчикам бота и печатает их
`getDescription()`. (Ему нужна ссылка на список обработчиков — см. ниже.)

> Если позже захочешь новую команду (например, «подбрось монетку») — просто
> добавишь ещё один такой класс. Это и есть «гибкость» из требований.

### Файл: `bot/IBot.java` — интерфейс бота

```java
public interface IBot {
    String respond(String userInput);          // главный метод для UI
    void registerHandler(MessageHandler h);     // добавить команду
    List<MessageHandler> getHandlers();         // для «помощи»
}
```
Зачем интерфейс: окно будет зависеть от `IBot`, а не от конкретного класса —
тогда логику можно переиспользовать (хоть в онлайн-боте).

### Файл: `bot/AbstractBot.java` — общий алгоритм

Абстрактный класс реализует `IBot` и пишет один раз главный цикл:
```java
public String respond(String input) {
    for (MessageHandler h : handlers)
        if (h.canHandle(input))
            return h.respond(input, history);
    return "Не понял. Напишите «помощь».";   // ответ по умолчанию
}
```
А какой набор обработчиков подключить — оставляет наследнику через абстрактный
метод `registerDefaultHandlers()`.

### Файл: `bot/SimpleBot.java` — конкретный бот

Реализует `registerDefaultHandlers()` и в нужном порядке добавляет обработчики:
```java
protected void registerDefaultHandlers() {
    registerHandler(new HelpHandler(this));
    registerHandler(new GreetingHandler());
    registerHandler(new TimeHandler());
    registerHandler(new StatsHandler());
    registerHandler(new CalculatorHandler());
}
```
Порядок = порядок проверки: специфичные раньше широких.

## Этап 4. ПРОВЕРКА логики БЕЗ окон (важный приём)

Перед тем как трогать UI, убедись, что бот работает. Сделай **временный**
консольный `main` (потом удалишь):

```java
public class ConsoleTest {
    public static void main(String[] args) {
        ChatHistory history = new ChatHistory();
        IBot bot = new SimpleBot(history);
        Scanner sc = new Scanner(System.in);
        while (true) {
            String line = sc.nextLine();
            if (line.equals("выход")) break;
            history.add(new Message(Author.USER, line));
            String answer = bot.respond(line);
            history.add(new Message(Author.BOT, answer));
            System.out.println("Бот: " + answer);
        }
    }
}
```
Проверь «Привет», «который час», «умножь 12 на 157», «статистика», «помощь».
Если в консоли всё отвечает правильно — **логика готова**, и UI её точно не
испортит. Только теперь переходим к окнам.

## Этап 5. UI через Scene Builder (FXML) — в самом конце

### Что такое FXML и контроллер

- **FXML** — XML-файл с разметкой окна (какие кнопки/поля где лежат). Его рисуют
  мышкой в **Scene Builder**, руками XML писать не нужно.
- **Контроллер** — Java-класс, привязанный к FXML. В нём:
  - поля с аннотацией `@FXML`, связанные с элементами по `fx:id`;
  - методы-обработчики, которые срабатывают на действия (нажатие кнопки и т.п.).

Связь задаётся в Scene Builder:
- свойство **Controller → fx:controller** = имя класса-контроллера;
- у элемента поле **fx:id** = имя поля в контроллере;
- у кнопки **On Action** = имя метода контроллера.

### Нам нужно ДВА окна

1. `login.fxml` + `LoginController` — окно ввода имени.
2. `chat.fxml` + `ChatController` — основное окно чата.

### Шаг 5.1. Окно входа

В Scene Builder сделай `login.fxml`: `TextField` (fx:id = `nameField`) и `Button`
(On Action = `onLogin`).

```java
public class LoginController {
    @FXML private TextField nameField;

    @FXML
    private void onLogin() {
        String name = nameField.getText().trim();
        if (name.isEmpty()) name = "Гость";
        openChatWindow(name);   // см. следующий шаг — передача данных
    }
}
```

### Шаг 5.2. Самое важное: КАК передать данные во второе окно

Это твой главный вопрос. Шаблон называется «получить контроллер у FXMLLoader и
вызвать его init-метод». Суть: мы **сами** загружаем второй FXML, после загрузки
у нас есть объект его контроллера, и мы кладём в него данные **до** показа окна.

В `ChatController` делаем метод-приёмник данных:
```java
public class ChatController {
    @FXML private ListView<Message> listView;
    @FXML private TextArea inputArea;

    private IBot bot;
    private ChatHistory history;
    private HistoryStorage storage;

    /** Этот метод вызывает окно входа и ПЕРЕДАЁТ сюда имя пользователя. */
    public void initData(String userName) {
        this.history = new ChatHistory();
        this.storage = new FileHistoryStorage("chat_history.txt");
        this.history.addAll(storage.load());   // загрузка истории при запуске
        this.bot = new SimpleBot(history);
        // ...отрисовать загруженную историю в listView...
    }
}
```

А в `LoginController.openChatWindow()` — собственно передача:
```java
private void openChatWindow(String name) {
    try {
        FXMLLoader loader = new FXMLLoader(getClass().getResource("/.../chat.fxml"));
        Parent root = loader.load();                 // 1) грузим разметку
        ChatController controller = loader.getController(); // 2) берём её контроллер
        controller.initData(name);                   // 3) ПЕРЕДАЁМ данные

        Stage stage = new Stage();
        stage.setScene(new Scene(root));
        stage.setTitle("Чат с ботом");
        stage.show();

        // закрыть окно входа
        ((Stage) nameField.getScene().getWindow()).close();
    } catch (IOException e) { e.printStackTrace(); }
}
```

**Запомни эти три строки** — это и есть «передача данных из окна в окно»:
```
Parent root = loader.load();
ChatController c = loader.getController();
c.initData(name);
```
Нельзя создавать контроллер через `new ChatController()` и потом грузить FXML —
FXMLLoader создаёт контроллер сам. Поэтому данные кладём именно так: загрузил →
взял контроллер → вызвал метод.

> Альтернатива на будущее: можно завести один общий объект-«состояние» приложения
> и отдавать его всем контроллерам. Но для двух окон способ выше — самый простой
> и понятный.

### Шаг 5.3. Основное окно: отправка сообщения

В `chat.fxml`: `ListView` (`fx:id=listView`), `TextArea` (`fx:id=inputArea`),
кнопка «Отправить» (`On Action = onSend`).

```java
@FXML
private void onSend() {
    String text = inputArea.getText().trim();
    if (text.isEmpty()) return;
    addMessage(new Message(Author.USER, text)); // в историю + на экран
    String answer = bot.respond(text);          // ← вся логика тут
    addMessage(new Message(Author.BOT, answer));
    inputArea.clear();
}
```
Заметь: окно НЕ знает, как бот думает — просто зовёт `bot.respond()`. Это и есть
независимость UI от логики.

### Шаг 5.4. Горячие клавиши (Enter / Ctrl+Enter)

Требование: отправка по Enter. На `inputArea` повесь обработчик нажатий —
в Scene Builder это свойство **On Key Pressed = onKeyPressed**, а в контроллере:
```java
@FXML
private void onKeyPressed(KeyEvent e) {
    if (e.getCode() == KeyCode.ENTER) {
        if (e.isShiftDown()) return;   // Shift+Enter — новая строка
        e.consume();                   // не вставлять перевод строки
        onSend();                      // Enter и Ctrl+Enter — отправить
    }
}
```

### Шаг 5.5. Главный класс приложения и сохранение при выходе

```java
public class ChatApplication extends Application {
    @Override
    public void start(Stage stage) throws IOException {
        FXMLLoader loader = new FXMLLoader(getClass().getResource("/.../login.fxml"));
        stage.setScene(new Scene(loader.load()));
        stage.setTitle("Вход");
        stage.show();
    }
    public static void main(String[] args) { launch(args); }
}
```

Сохранение истории при закрытии вешается на главное окно чата
(в `ChatController.initData` после получения `stage`, либо в `Application.stop()`):
```java
stage.setOnCloseRequest(e -> storage.save(history.getMessages()));
```
